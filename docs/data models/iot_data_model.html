Time Series Data
sensor:<id>:readings (Sorted Set)
  • Member: JSON reading
  • Score: Unix Timestamp
# import.py:155-165 reading = json.dumps({ "temp": round(temperature, 2), "humidity": round(humidity, 2), "light": round(light, 2), "voltage": round(voltage, 3), "date": date_str, "time": time_str }) r.zadd(f"sensor:{mote_id}:readings", {reading: epoch})

Delete Old Readings:
# queries.py:156-159
removed = r.zremrangebyscore(

)
Why Sorted Set?
→ Score = timestamp for ordering
→ ZRANGEBYSCORE for time windows
→ O(log N + M) range queries
→ No RedisTimeSeries needed
